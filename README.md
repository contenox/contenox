**Local AI that actually *does* stuff on your machine.**

Contenox is a lightning-fast, fully-local CLI that turns natural language goals into **persistent, step-by-step plans** and executes them with real shell + filesystem tools. Powered by any LLM that supports tool calling (Ollama, OpenAI, Gemini, vLLM). Zero cloud required.

```bash
$ contenox plan new "install a git pre-commit hook that prevents commits when go build fails"
Creating plan "install-a-git-pre-commit-a3f9e12b" with 5 steps. Now active.

$ contenox plan next --auto
Executing Step 1: Install necessary tools...              âœ“
Executing Step 2: Create .git/hooks/pre-commit...         âœ“
Executing Step 3: Edit the hook script with the check...  âœ“
Executing Step 4: Write bash content to the hook file...  âœ“
Executing Step 5: chmod +x .git/hooks/pre-commit...       âœ“

No pending steps. Plan is complete!
```

**The model wrote that hook.** On *your* machine. No copy-paste hell.

## Why Contenox?

Most AI coding agents are chatty one-shots. Contenox is different:

- **Persistent plans** stored in SQLite â€” pause, inspect, retry, replan at any time
- **Human-in-the-loop by default** â€” `--auto` only when you say so
- **Real tools** â€” shell commands and filesystem, not just code suggestions
- **Fully offline** with Ollama â€” no data leaves your machine
- **Chains are just JSON** â€” write your own LLM workflows

## Quick Start

### Install

**Ubuntu / Linux**
```bash
TAG=$(curl -sL https://api.github.com/repos/contenox/contenox/releases/latest | grep '"tag_name"' | cut -d'"' -f4)
ARCH=$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
curl -sL "https://github.com/contenox/contenox/releases/download/${TAG}/contenox-${TAG}-linux-${ARCH}" -o contenox
chmod +x contenox && sudo mv contenox /usr/local/bin/contenox
contenox --version
```

**macOS**
```bash
TAG=$(curl -sL https://api.github.com/repos/contenox/contenox/releases/latest | grep '"tag_name"' | cut -d'"' -f4)
ARCH=$(uname -m | sed 's/x86_64/amd64/;s/arm64/arm64/')
```

Or pick a binary from [Releases](https://github.com/contenox/contenox/releases).

### First run

```bash
contenox init                              # scaffold .contenox/ with config + chain
contenox "list files in my home directory" # instant agentic chat
```

**Requirements:** an LLM with tool calling support.
*Local:* `ollama serve && ollama pull qwen2.5:7b`
*Cloud:* set your API key in `.contenox/config.yaml` (generated by `contenox init`).

## `contenox plan` â€” The Killer Feature

```bash
contenox plan new "migrate all TODO comments in the codebase to TODOS.md"
contenox plan new "set up a git pre-commit hook that blocks commits when go build fails"
contenox plan new "find all .go files larger than 500 lines and write a refactoring report"
```

Contenox breaks any goal into an ordered plan, then executes it step by step using real tools.

### Commands

| Command | What it does |
|---|---|
| `contenox plan next` | Run **one** step (safe default â€” review before continuing) |
| `contenox plan next --auto` | Run **all** remaining steps autonomously |
| `contenox plan show` | See the active plan + step status |
| `contenox plan list` | All plans (`*` = active) |
| `contenox plan retry <N>` | Re-run a failed step |
| `contenox plan skip <N>` | Mark a step skipped and move on |
| `contenox plan replan` | Let the model rewrite the remaining steps |
| `contenox plan delete <name>` | Delete a plan by name |
| `contenox plan clean` | Delete all completed plans |

**Pro tip:** Always do `contenox plan show` before `--auto`.

### Full example

```bash
# 1. Create
contenox plan new "install a git pre-commit hook that blocks commits when go build fails"

# 2. Review the plan before touching anything
contenox plan show

# 3. Execute one step at a time
contenox plan next
contenox plan next
# ...

# Or run everything at once once you trust it
contenox plan next --auto

# 4. If a step went wrong
contenox plan retry 3

# 5. Final check
contenox plan show
```

## Other Modes

### `contenox` â€” Interactive chat

```bash
contenox "what is my current working directory?"
contenox list files in my home directory
contenox --input "explain this file"
echo "explain this" | contenox
```

Uses `.contenox/default-chain.json`. Natural language â†’ shell tools â†’ response.

### `contenox exec` â€” Scriptable, stateless execution

For CI/pipelines where you want full control:

```bash
contenox exec --chain .contenox/my-chain.json "what is 2+2?"
cat diff.txt | contenox exec --chain .contenox/review.json --input-type chat
contenox exec --chain .contenox/doc-chain.json --input @main.go
contenox exec --chain .contenox/parse-chain.json --input-type json '{"key":"value"}'
```

`exec` is stateless â€” no chat history. `--chain` is required. Supported `--input-type`: `string` (default), `chat`, `json`, `int`, `float`, `bool`.

## Configuration (`.contenox/config.yaml`)

`contenox init` generates this. Edit to select your provider.

**Local (Ollama â€” default)**
```yaml
backends:
  - name: local
    type: ollama
    base_url: http://127.0.0.1:11434
default_provider: local
default_model: qwen2.5:7b
context: 32768
enable_local_shell: true
local_shell_allowed_commands: "bash,echo,cat,ls,chmod,sh,date,pwd,head,tail,grep,find,mkdir,cp,mv"
```

**OpenAI**
```yaml
backends:
  - name: openai
    type: openai
    base_url: https://api.openai.com/v1
    api_key_from_env: OPENAI_API_KEY
default_provider: openai
default_model: gpt-4o-mini
```

**Gemini**
```yaml
backends:
  - name: gemini
    type: gemini
    api_key_from_env: GEMINI_API_KEY
default_provider: gemini
default_model: gemini-2.0-flash
```

| Backend | `type` | Notes |
|---|---|---|
| Ollama | `ollama` | Local. `ollama serve` first. |
| OpenAI | `openai` | `api_key_from_env` or `api_key` |
| Gemini | `gemini` | `api_key_from_env` or `api_key` |
| vLLM | `vllm` | Self-hosted OpenAI-compatible endpoint |

## Safety

- **Opt-in shell access** â€” `enable_local_exec` is false by default
- **Whitelist** â€” only commands you explicitly allow can run
- **Human-in-the-loop** â€” `plan next` executes one step and stops; `--auto` requires explicit intent
- **Local-first** â€” with Ollama, nothing leaves your machine

## Architecture

```

  â”œâ”€â”€ plan new      â†’ LLM planner chain â†’ SQLite plan + steps
  â”œâ”€â”€ plan next     â†’ LLM executor chain â†’ local_shell / local_fs â†’ result persisted
  â”œâ”€â”€ exec          â†’ run any chain, any input type, stateless

  â”œâ”€â”€ message_index        (chat sessions)
```

Chains are JSON files in `.contenox/`. They define the LLM workflow: model, hooks, branching logic. See [ARCHITECTURE.md](ARCHITECTURE.md) for the full picture.

Contenox is also the local CLI layer of a full **Runtime API** server (PostgreSQL + NATS) for production deployments. [Read the server docs â†’](docs/server-quickstart.md)
